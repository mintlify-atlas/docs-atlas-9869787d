---
title: "Health check"
description: "Check whether the Airbyte API is available."
---

The health endpoint returns a simple `200 OK` response when the API is operational. You can use it to verify connectivity or integrate into monitoring systems.

## Endpoint

| Method | Path | Description |
|---|---|---|
| `GET` | `/v1/health` | Returns 200 when the API is healthy |

---

## Health check

<CodeGroup>
```bash curl
curl https://api.airbyte.com/v1/health
```
</CodeGroup>

`GET /v1/health`

This endpoint does not require authentication.

### Response

Returns HTTP `200 OK` with an empty body when the API is healthy.

```bash
# Successful response
HTTP/2 200
```

If the API is unavailable or degraded, you will receive a non-`2xx` status code.

<Tip>
Use the health endpoint in readiness probes or uptime monitors to detect API outages early.
</Tip>
