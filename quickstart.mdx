This guide walks you through running Airbyte on your local machine using Docker Compose. By the end you will have the full platform running and your first data connection created.

## Prerequisites

Before you start, make sure you have the following installed:

- **Docker** 20.10.0 or later ([install Docker](https://docs.docker.com/get-docker/))
- **Docker Compose** v2.0.0 or later (bundled with Docker Desktop; verify with `docker compose version`)
- At least **4 GB of RAM** allocated to Docker
- **Git**

<Warning>
  On macOS, open **Docker Desktop → Settings → Resources** and set memory to at least 4 GB before proceeding.
</Warning>

## Run Airbyte

<Steps>
  <Step title="Clone the repository">
    Clone the Airbyte monorepo and move into it:

    ```bash
    git clone https://github.com/airbytehq/airbyte.git
    cd airbyte
    ```
  </Step>

  <Step title="Start all services">
    Run the helper script (or `docker compose` directly) to pull images and start every service:

    ```bash
    ./run-ab-platform.sh
    ```

    Alternatively, start the stack directly with Docker Compose:

    ```bash
    docker compose up -d
    ```

    On first run Docker pulls all container images. This typically takes 2–5 minutes depending on your connection speed.
  </Step>

  <Step title="Wait for the platform to become healthy">
    Watch the bootloader complete database migrations and the server come online:

    ```bash
    docker compose logs -f airbyte-server
    ```

    Wait until you see a line similar to:

    ```
    INFO  i.a.s.ServerApp - Starting Airbyte Server
    ```

    The full startup sequence — bootloader migrations, Temporal worker registration, server readiness — takes roughly 30–60 seconds on a modern laptop.
  </Step>

  <Step title="Open the web UI">
    Navigate to [http://localhost:8000](http://localhost:8000) in your browser.

    The default credentials for a fresh local install are:

    | Field | Value |
    |-------|-------|
    | **Username** | `airbyte` |
    | **Password** | `password` |
  </Step>

  <Step title="Create your first connection">
    1. In the left sidebar, click **Sources** and then **New source**.
    2. Search for and select a source connector (for example, **GitHub** or **Faker** for a zero-credential test).
    3. Fill in the required credentials and click **Set up source**.
    4. Click **Destinations** → **New destination** and configure where you want the data to land (for example, **Local JSON** for a quick local test).
    5. Click **Connections** → **New connection**, select the source and destination you just created, choose a sync frequency, and click **Set up connection**.
    6. Click **Sync now** to run your first sync.

    <Tip>
      Use the **Faker** source connector to generate synthetic data without needing any external API credentials. Pair it with the **Local JSON** destination to see records land immediately on disk.
    </Tip>
  </Step>
</Steps>

## Verify the sync succeeded

After the sync completes, the **Connections** page shows a green status badge and the number of records synced. Click the connection name to see per-stream stats and logs.

## Stopping Airbyte

To stop all containers while preserving your data:

```bash
docker compose down
```

To stop and remove all data volumes (full reset):

```bash
docker compose down -v
```
